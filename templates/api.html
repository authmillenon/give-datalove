give.datalove API

Why an API?

We encourage everybody to participate in the growth of give.datalove. You may use this API to build up your own widgets or datalove sites.

How do I use the API?

It is rather easy. We have some URLs to call to get back certain information.

http://give.datalove.me/api/users

Gives back all users of give.datalove.me with their received datalove, in random order.

HTTP Status of the Response:

  • 200 OK: All went right. The message's text is the list of users with their received love in random order in an comma seperated value format.
  • 500 Internal Server Error: There is an error at our end of the line. The message's text is an error message and the traceback of the error. If you want to help make our little project better, report the error to https://github.com/authmillenon/give-datalove/issues.

http://give.datalove.me/api/$$user

Gives back the profile information (username, received datalove, websites) of the user named $$user.

HTTP Status of the Response:

  • 200 OK: All went right. The message's text is the user's available love.
  • 500 Internal Server Error: There is an error at our end of the line. The message's text is an error message and the traceback of the error. If you want to help make our little project better, report the error to https://github.com/authmillenon/give-datalove/issues.

http://give.datalove.me/api/$$user/available_datalove

Gives back the available datalove to share of the user named $$user.

HTTP Status of the Response:

  • 200 OK: All went right. The message's text is the user's available love.
  • 404 Not Found: There is no user called $$user in the database. The message's text is an error message.
  • 500 Internal Server Error: There is an error at our end of the line. The message's text is an error message and the traceback of the error. If you want to help make our little project better, report the error to https://github.com/authmillenon/give-datalove/issues.

http://give.datalove.me/api/login

Logging in a user, i. e. associating him with a session on the server. This request must be send as a POST request with nickname and password in the query.

HTTP Status of the Response:

  • 200 OK: All went right. The message's text is the user's new session ID.
  • 400 Bad Request: The $$nickname and $$password are not associated. The message's text is an error message.
  • 401 Unauthorized: The server got an illegal session ID. The message's text is an error message.
  • 404 Not Found: You have tried to use GET to log in. You must use POST
  • 412 Precondition Failed: Some preondition was not met. The message's text is an error message describing what to do.
  • 500 Internal Server Error: There is an error at our end of the line. The message's text is an error message and the traceback of the error. If you want to help make our little project better, report the error to https://github.com/authmillenon/give-datalove/issues.

http://give.datalove.me/api/$$user/give_datalove?sid=$$session_id

will give datalove to this user $$user

Giving a user named $$user 1 datalove. This request needs the giving user to be logged in and his session ID $$session_id must be in the query as shown. Alternatively you can send the session ID send as a cookie called 'give_datalove_session_id'

HTTP Status of the Response:

  • 200 OK: All went right. The message's text is the user's new session ID.
  • 401 Unauthorized: The session ID is not associated to a user, i. e. the user is not logged in. The message's text is an error message.
  • 404 Not Found: There is no user called $$user in the database. The message's text is an error message.
  • 412 Precondition Failed: Some preondition was not met. The message's text is an error message describing what to do.
  • 500 Internal Server Error: There is an error at our end of the line. The message's text is an error message and the traceback of the error. If you want to help make our little project better, report the error to https://github.com/authmillenon/give-datalove/issues.
